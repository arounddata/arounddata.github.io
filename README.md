# Вычисление замыкания системы функциональных зависимостей


---

## Введение

Программа вычисляет замыкание системы функциональных зависимостей (ФЗ) с использованием правил транзитивности и псевдотранзитивности.

Алгоритм основан на оригинальном PL/I коде (1986) и адаптирован для веб-интерфейса.

---

## Формат ввода

ФЗ вводятся в левой панели в **строковой форме** (с-форма):

- `A-B` — атрибут A определяет B
- `A-B-C` — атрибут A определяет B и C (две ФЗ: A-B, A-C)
- `A*B-C` — атрибуты A и B совместно определяют C
- `A*B*C-D-E-F` — атрибуты A, B и C совместно определяют D, E и F (`A*B*C-D`,`A*B*C-E`,`A*B*C-F`)

**Разрешённые символы в именах атрибутов:** любые, кроме `*`, `-` и пробела.

## Управление
| Кнопка | Горячая клавиша | Действие |
|--------|-----------------|----------|
| Открыть | Ctrl+O | Загрузить XML-файл |
| Проверить | Ctrl+K | Преобразовать с-форму в ч-форму |
| Рассчитать | Ctrl+R | Вычислить замыкание |
| Сохранить | Ctrl+Shift+S | Сохранить результат в XML |
| Справка | F1 | Открыть это окно |

## Схема работы кнопки «Открыть»:

```
1. Очистка всех панелей и состояния
2. Открытие диалога выбора файла
3. Чтение и парсинг файла
    3.1. Чтение файла
    3.2. Удаление <fdsc> и комментариев
    3.3. Поиск тегов <fd...>
    3.4. Проверка наличия ФЗ
4. Загрузка данных в appState (originalFds = fdsList)
5. Обновление интерфейса
    5.1 Отображение данных в левой панели
    5.2 Активация кнопки «Проверить»
    5.3 Деактивация «Рассчитать» и «Сохранить»
6. Обновление статусной строки и информационной панели
7. Ожидание действий пользователя
```

## Формат файла

XML-файл должен содержать теги `<fd...>`:

```xml
<fdsi>
    <fd1>A-B-C</fd1>
    <fd2>A*B-C-D</fd2>
</fdsi>
```
конец справки
